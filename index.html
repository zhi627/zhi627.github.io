auto-register

本文将介绍zabbix中自动注册流程,个人觉得auto-registion对自动化运维有较大的帮助。zabbix-agentd自动向zabbix-server注册主机，
同时可以设置其自动注册时条件以及对应的操作，

自动注册

主动模式使用的,在”配置”(Configuration)中找到”Actions”(动作),选择”事件源”(Event source)中的”自动注册”(Auto Registion),并且填写正常的信息

  • 在表单导航行选择”动作”并且填写相应信息,比如”名称”、”默认接收人”、”默认信息”、
  • 导航栏中选择”条件”，新的触发条件，触发条件有三种:
    • agent代理程序(Proxy)
    • 主机元数据(Host metadata)
    • 主机名称(Host name)
  • 在操作中，我们可以有几种操作类型
    • 发送消息
    • 远程命令
    • 添加到主机
    • 添加到主机群组
    • 与模板关联
    • 停用主机
    • 设置主机清单模式

例子

  • 方法1：设置HostMetadata
    我们在zabbix-server端创建一个主机自动添加到linux主机群组的自动注册。
    设置”主机元数据” 似 linux，并且在操作中设置添加主机,添加到主机群组”Linux Servers”,同时设置关联Linux通过类型为zabbix_agentd(active)方式的监控项的模板。
    同时,zabbix-agent端要设置主动模式,并且conf文件设置HostMetadata = linux,值是自己定义的
    这样，当第一次zabbix agentd启动时，主动将数据提交到zabbix server。zabbix server 根据自动注册规则将主机添加到”Linux Servers”主机群组，并且自动管理了Linux 模板。
  • 方法2:设置HostMetadataItem
    zabbix-server端的配置与方法1一致。但是在zabbix-agent设置主动模式,并且conf文件设置 HostMetadataItem=system.uname,值来之key

应用场景:

设置多个自动注册动作，在agent的conf文件设置HostMetadata多个关键字，根据关键字自动添加到相应主机群组以及管理相应模板

例子：

conf文件设置Linux MySQL等关键字，HostMetadata=Linux MySQL。同时zabbix server端分别设置自动注册Linux群组并且关联Linux模板，自动注册MySQL群组并且关联MySQL监控模板。当主机自动注册时，关联到Linux和MySQL的模板。当主机2是需要监控Linux和Redis时，我们只需要再增加一个关于Redis的自动注册动作，主机2的conference文件HostMetadata=Linux Redis即可。这样使监控更为灵活以及更加自动化

zabbix 简介

zabbix,简单来说是一款企业级、开源、分布式的基于web界面的监控系统。能监视各种网络参数、程序性能信息、系统信息。可通过邮件、自定义脚本（微信公众号推送、短信推送、第三方推送）实现将zabbix的报警信息推送到用户上。

zabbix主要由两部分组成，zabbix server与可选组件zabbix agent。
zabbix server可以通过SNMP,zabbix agent,ipmi,jmx等方法对系统、网络、程序进行数据收集。

特点：

  • 分布式监控。具有节点(node)模式和代理(proxy)模式。
  • 自动化功能。自动发现，自动注册功能。在部署监控中，可以让服务器需要监控的信息自动注册到server,对相应监控项进行监控。使用ansible或者saltstack可轻松实现自动化监控
  • 模块化监控。设置监控模板，监控部署更为方便。
  • 自定义监控以及通知。 除了监控系统信息，还可以自定义监控网络，程序等，自定义通知脚本，通知文本格式
  • 支持多种方式监控。agentd、snmp、ipmi、jmx等模式
  • 多种安全认证方式。
  • 提供api功能。方便二次开发
  • 易操作。基于web界面添加相关监控

安装相关软件

  • httpd以及扩展需要的库

安装mysql

  • 根据Centos安装mysql官方yum源

  • 安装mysql

安装php

  • 编译安装php以及相关扩展

安装zabbix

  • 安装zabbix官方yum源

  • 安装数据库文件以及前端文件

  • 安装zabbix相关程序

创建并导入数据库文件

  • 创建zabbix数据库,mysql5.7默认需要设置更为复杂的秘密

  • 导入数据库文件,记住zabbix-server-mysql-3.0.X,是对应自己安装zabbix的版本

设置虚拟主机

  • 根据httpd版本，可能会有不同语法，根据情况修改配置文件，常见是httpd2.2与httpd2.4的区别
    Alias /zabbix /usr/share/zabbix
<Directory "/usr/share/zabbix">
    Options FollowSymLinks
    AllowOverride None
    Allow from all
    <IfModule mod_php5.c>
        php_value max_execution_time 300
        php_value memory_limit 128M
        php_value post_max_size 16M
        php_value upload_max_filesize 2M
        php_value max_input_time 300
        php_value always_populate_raw_post_data -1
        php_value date.timezone PRC
     </IfModule>
</Directory>

配置数据库以及登录

  • 打开浏览器,访问http://hostname/zabbix/setup.php, 填写mysql相关信息,host填写为127.0.0.1以及检测软件是否满足要求
  • 编辑/etc/zabbix/zabbix_server.conf文件设置mysql数据库信息
  • service zabbix-server start
  • 登录，默认账号:Admin,密码:zabbix

优化以及配置

  • 根据监控情况进行相关配置，如果监控主机数较多，需要配置mysql的max-connections选项
  • 监控方式zabbix_agentd主动模式多的话,需要配置Trapper数量
  • 监控方式zabbix_agentd被动方式多的话,需要配置Poller数量
  • 当监控主机数量较多,建议使用主动模式或者proxy模式

Hexo与GiHub Pages 搭建个人博客

本文将详细介绍如何在 Centos 系统使用 Hexo 以及GitHub Pages 搭建个人博客

注册GitHub账号

安装Git

安装Node.js以及npm

在国内用户可以更新yum源以及npm源为国内的，这样下载安装更快，可参考 yum源更换 以及 npm源更换

安装Hexo

用Hexo创建blog

测试Hexo是否可以运行

  • 安装hexo-server插件，可将服务器独立成个别模块
  • 启动服务器，提示博客会在http://localhost:4000/下启动。 在服务器启动期间，Hexo会监视文件变动并自动更新，无须重启服务器。不断刷新网页即可看到更新。

创建ssh私钥与公钥

  • 调用ssh-keygen生成对应用户的私钥与公钥文件id_rsa与id_rsa.pub,一路回车即可
  • 在自己创建的GitHub Pages项目Settings找到Deploy keys,并复制id_rsa.pub公钥内容到Key文本框中
  • 勾选Allow write access复选框,设置可以通过密钥方式修改博客内容

设置git账号

  • 设置为自己GitHub账号以及邮箱

设置hexo提交更新到GitHub

  • 设置hexo创建的博客文件夹的_config.yml中deploy设置为git部署
    deploy:

    type: git
repo: git@github.com:account/blogname.github.io.git
branch: master

  • 安装hexo-deployer-git插件可以使用git进行部署

  • 生成静态文件并且提交到GitHub
