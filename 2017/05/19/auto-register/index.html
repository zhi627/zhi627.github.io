auto-register

本文将介绍zabbix中自动注册流程,个人觉得auto-registion对自动化运维有较大的帮助。zabbix-agentd自动向zabbix-server注册主机，
同时可以设置其自动注册时条件以及对应的操作，

自动注册

主动模式使用的,在”配置”(Configuration)中找到”Actions”(动作),选择”事件源”(Event source)中的”自动注册”(Auto Registion),并且填写正常的信息

  • 在表单导航行选择”动作”并且填写相应信息,比如”名称”、”默认接收人”、”默认信息”、
  • 导航栏中选择”条件”，新的触发条件，触发条件有三种:
    • agent代理程序(Proxy)
    • 主机元数据(Host metadata)
    • 主机名称(Host name)
  • 在操作中，我们可以有几种操作类型
    • 发送消息
    • 远程命令
    • 添加到主机
    • 添加到主机群组
    • 与模板关联
    • 停用主机
    • 设置主机清单模式

例子

  • 方法1：设置HostMetadata
    我们在zabbix-server端创建一个主机自动添加到linux主机群组的自动注册。
    设置”主机元数据” 似 linux，并且在操作中设置添加主机,添加到主机群组”Linux Servers”,同时设置关联Linux通过类型为zabbix_agentd(active)方式的监控项的模板。
    同时,zabbix-agent端要设置主动模式,并且conf文件设置HostMetadata = linux,值是自己定义的
    这样，当第一次zabbix agentd启动时，主动将数据提交到zabbix server。zabbix server 根据自动注册规则将主机添加到”Linux Servers”主机群组，并且自动管理了Linux 模板。
  • 方法2:设置HostMetadataItem
    zabbix-server端的配置与方法1一致。但是在zabbix-agent设置主动模式,并且conf文件设置 HostMetadataItem=system.uname,值来之key

应用场景:

设置多个自动注册动作，在agent的conf文件设置HostMetadata多个关键字，根据关键字自动添加到相应主机群组以及管理相应模板

例子：

conf文件设置Linux MySQL等关键字，HostMetadata=Linux MySQL。同时zabbix server端分别设置自动注册Linux群组并且关联Linux模板，自动注册MySQL群组并且关联MySQL监控模板。当主机自动注册时，关联到Linux和MySQL的模板。当主机2是需要监控Linux和Redis时，我们只需要再增加一个关于Redis的自动注册动作，主机2的conference文件HostMetadata=Linux Redis即可。这样使监控更为灵活以及更加自动化
