Linux 文件系统

发表于 2017-07-03 | 分类于 shell | 阅读次数

简介

Linux有多种文件系统，下面简单讲解如何在命令行上创建和管理它们。

文件系统

ext文件系统

ext文件系统又叫扩展文件系统。使用虚拟目录来操作硬件设备，在物理设备上按定长的块存储数据。
ext文件系统采用称作为索引节点的系统来存放虚拟目录中所存储文件的信息。索引节点系统在每个物理设备中创建一个但难度的表（称为索引节点表）来存储这些文件的信息。存储在虚拟目录中的每一个文件在索引节点表中都有一个条目。条目名称的扩展部分来自其跟踪每个文件的额外数据:

  • 文件名；
  • 文件大小
  • 文件的属主
  • 文件的属组
  • 文件的访问权限
  • 指向存有文件数据的每个硬盘块的指针

Linux通过唯一的数值来引用索引节点表中的每个索引节点，这个值是创建文件时由文件系统分配的。文件系统通过索引节点号而不是文件全名及路径来标识文件。

ext2文件系统

最早的ext文件系统有不少限制，比如文件大小不得超过2GB。是ext文件系统基本功能的一个扩展，但是维护着同样的结果，最大文件大小增加到2TB,后期版本增加到32TB
索引节点表为文件添加了：

  • 创建时间值
  • 修改时间值
  • 最后访问时间值
    ext2文件系统通过按组分配磁盘块来减轻碎片化。通过将数据块分组，文件系统不需要为了数据块查找整个无聊设备来读取文件。

日志文件系统

日志文件系统增加了一层安全性。取代了之前先将数据直接写入存储设备再更新索引节点表的做法，日志文件系统会将文件的更改写入到历史文件（称为日志,journal）中，然后在数据成功写到存储设备和索引节点表之后，再删除对应的日志条目。

3种不同的广泛使用的日志方法，
|方法|描述|
|-|-|
|数据模式|索引节点和文件都会被写入到日志；丢失数据风险低，但性能差|
|排序模式|只有索引节点数据会被写入日志但只有数据成功写入后才删除；性能和安全之间的良好折中|
|会写模式|只有索引节点数据会被写入日志，但不管文件数据合适写入，丢失数据风险高，但仍比不用日志好|

ext3文件系统

ext3采用和ext2文件系统相同的索引结点表结构，但给每个存储设备增加了一个日志文件，来将准备写入存储设备的数据线写进日志文件。
默认情况下，ext3文件系统用排序模式的日志功能—只将索引节点信息写入日志问问你见，直到数据块都被成功写入存储设备才删除。
ext3为Linux文件系统添加了基本的日志功能，但他仍然缺一些东西。例如ext3文件系统无法恢复误删的文件，它没有任何内建的数据压缩功能(虽然有个需单独安装的补丁支持这个功能)，ext3文件系统也不支持加密文件

ext4文件系统

ext4文件系统在2008年时被Linux内核官方支持处理支持数据压缩和加密，还支持一个称作为区段的特性。区段在粗出设备上按块分配空间，但在索引节点表中只保存起始块的位置。
ext4还整合了块预分配。如果你想在存储设备上给一个你知道变大的文件预留空间啊，通过ext4文件系统你可以为文件分配所有期望的块，不只是物理上存在的块。ext4文件系统用0填满预留的数据块，并知道不要将它们分配给其他文件。

Reiser文件系统

Reiser文件系统只支持回写模式–只把索引节点表数据写道日志文件。ReiserFS文件系统是Linux上最快的日志子文件系统之一。
Reiser文件系统有两个有意思的特性：

  1. 可以在线调整已有文件系统的大小
  2. 尾部压缩的技术，该技术能将一个文件的数据填进另一个文件的数据块中的空白空间。

JFS文件系统

采用顺序日志方法，即只在日志中保存索引节点表书，直到真正的文件数据被写进存储设备时才删除它。同时，采用基于区段的文件分配，即为每个写入存储设备的分配一组块。

XFS文件系统

XPFS文件系统采用回写模式的日后，还允许在线调整文件系统的大小，但是XFS文件系统只可扩大不能缩小。

操作文件系统

fdisk

命令 描述
a 设置一个标识，说明这个分区是可启动的
b 编辑BSD Unix系统用的磁盘标签
c 设置 DOS 兼容标识
d 删除分区
l 显示可用的分区类型
m 显示命令选项
n 添加一个新分区
o 创建DOS分区表
p 显示当前分区表
q 退出，不保存更新
s 为Sun Unix 系统创建一个新磁盘标签
t 修改分区的系统ID
u 改变使用的存储单位
v 验证分区表
w 将分区表写入磁盘
x 高级功能

分区可以按主分区或扩展分区创建。主分区可以被文件系统直接格式化，而扩展分区只能容纳其他主分区。

fsck命令

fsck命令用来检查和修复任意类型的Linu文件系统，包括本章早些时候讨论过的所有文件系统-ext、ext2\ext3、ext4、ReiserFS、JFS和XFS。
命令格式：
fsck options filesystem

fsck的命令行选项
|选项|描述|
|-|-|
|-a|如果检测到错误，自动修复文件系统|
|-A|检查/etc/fstab文件中列出的所有文件系统|
|-C|给支持进度条功能的文件系统显示一个进度条(只有ext2和ext3)|
|-N|不进行检查，只显示哪些检查会执行|
|-r|出现错误时提示|
|-R|使用-A选项时跳过根文件系统|
|-s|检查多个文件系统时，依次进行检查|
|-t|指定要检查的文件系统类型|
|-T|启动时不显示信息|
|-V|在检查时产生详细输出|
|-y|检测到错误时自动修复文件系统|

注意

你只能在未挂载的文件系统上运行fsck命令。对大多数文件系统来说，你只需卸载文件系统来检查，检查完成了重新挂载就好了。但因为根文件系统含有所有核
心Linux命令和日志文件，你不能在运行的系统上卸载它。只需用LiveCD启动系统就可以了，然后在根文件系统上运行fsck命令。

逻辑卷管理器

逻辑卷管理的河西是它如何处理安装在系统上的硬盘分区。在逻辑卷管理的世界里，硬盘称作物理卷。每个物理卷都会映射到硬盘上创建的某一物理分区。

Linux 的LVM

  • LVM1：基本的逻辑卷管理功能。
  • LVM2: 除了标准的逻辑卷管理功能外，允许创建可读写快照、条带化、镜像

使用Linux LVM

  1. 定义物理卷，可以通过pvcreate来创建物理卷
  2. 创建卷组，可以通过vgcreate命令，vgdisplay命令查看新创建的卷组的细节
  3. 创建逻辑卷，lvcreate命令。
