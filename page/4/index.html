zabbix低级别发现

发表于 2017-05-25 | 分类于 zabbix | 阅读次数

简介

zabbix低级别发现(LLD)在单机多实例的情况下比较有用,很多情况使用netstat -nlp获取绑定ip以及端口的服务程序。
例如:MySQL 、Redis 等常用server端程序。或者直接获取服务端口只检测端口是否可到达的时候十分方便。以及设置相关监控项、触发器、图表等,
省去人工添加模板的步骤。当然还是要创建相关LLD的模板
主要注意几点:

  1. 是否开启了ipv6地址允许,可见为什么 netstat 对某些服务只显示了 tcp6 监听端口,是否需要禁用ipv6,根据情况处理LLD脚本
  2. 是使用UDP还是TCP协议的服务程序
  3. 使用netstat的权限问题,默认用户是zabbix,运行权限需要是root,设置sudo规则
  4. zabbix是nologin用户需要注释sudo配置文件中的Defaults requiretty

zabbix-server 配置文件详解

发表于 2017-05-24 | 分类于 zabbix | 阅读次数

简介

主要描述下server与agent的不同配置项
主要有几点需要注意:

  1. 当agent主动模式监控主机较多时,应提高StartTrappers,当被动模式监控主机较多时,应提高StartPollers
  2. 当监控主机较多时,并且使用MySQL时,应提高MySQL配置文件的max_connections的值
  3. yum安装zabbix-server的运行用户是zabbix,AlertScriptsPath应让zabbix有权限执行
  4. 安装完,需要配置zabbix_server.conf的数据库连接配置

zabbix-agent配置文件详解

发表于 2017-05-22 | 分类于 zabbix | 阅读次数

简介

主要描述下zabbix-agent的配置文件的相关配置项
有几点需要注意下:

  1. 如果需要调试一般设置DebugLevel=4,并且重启agent,一般可以看到足够的调试信息,包括脚本调用的错误信息,server与agent通信的信息
  2. 如果网络不畅通,出现server端接收数据不完整等,可以尝试设置高点超时时间,最大为30s
  3. 如果要接受server端的远程命令那就要设置Server以及StartAgents配置项且StartAgents要大于0
  4. 主动检测模式Hostname配置项在server端的要全局唯一
  5. UserParameter的shell command,AllowRoot要注意权限问题以及其key接受参数是否接受特殊字符
